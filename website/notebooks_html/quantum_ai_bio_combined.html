Abstract¶

We presentan end-to-end demonstration of a hybrid quantum–classical computational pipeline that unifies three research frontiers:

  1. QAOA (Quantum Approximate Optimization Algorithm) — a variational quantum circuit that encodes combinatorial optimisation problems (MaxCut) in a Hamiltonian and finds approximate solutions on $n \leq 12$ qubits via a classical outer loop.
  2. GNN-based QAOA warm-starting — a Graph Convolutional Network that replaces per-instance Nelder–Mead optimisation with a single microsecond forward pass, achieving >95% of the classical optimum after end-to-end training.
  3. Biomedical GCN classification — a two-layer GCN applied to a k-NN similarity graph of the Wisconsin Breast Cancer Dataset (WDBC), demonstrating that graph-structured inductive biases improve diagnostic accuracy over flat MLPs.

All components produce artefacts (edge lists, optimised angles, processed CSVs) saved to outputs/ for reproducibility.

0. Theoretical Foundation: Why Hybrid Quantum–Classical? ¶

The NISQ Era and Its Constraints¶

We live in the Noisy Intermediate-Scale Quantum (NISQ) era (Preskill, 2018): quantum computers with 50–1000 qubits exist, but gate errors (~0.1–1%), decoherence, and limited connectivity prevent fault-tolerant computation. NISQ algorithms must:

  • Use short circuits (few gates) to stay within the coherence time
  • Be noise-resilient (small errors don't catastrophically affect the result)
  • Delegate the hardest parts to classical hardware

QAOA is designed precisely for this regime: it uses circuits of constant depth $p$ (gate count $O(p \cdot m)$) and alternates between short quantum evolutions and classical parameter updates.

The Quantum–Classical Feedback Loop¶

         ┌─────────────────────────────────┐
         │  Classical Computer              │
         │  ┌──────────────────────────┐    │
         │  │  Optimise (γ, β)         │    │
         │  │  Nelder-Mead / Adam / GNN│    │
         │  └───────────┬──────────────┘    │
         └──────────────┼────────────────────┘
                        │  send (γ, β)
         ┌──────────────┼────────────────────┐
         │  Quantum Computer                  │
         │  ┌───────────▼──────────────┐      │
         │  │  Prepare |γ,β⟩ circuit  │      │
         │  │  Measure ⟨C⟩            │      │
         │  └───────────┬──────────────┘      │
         └──────────────┼────────────────────┘
                        │  return ⟨C⟩
                        └──► update (γ, β)

The GNN replaces the entire classical loop with a single feedforward pass: given the graph structure, it directly outputs $(\hat\gamma, \hat\beta)$ that approximate the optimal angles.

Mathematical Framework: Variational Quantum Algorithms¶

QAOA belongs to the class of Variational Quantum Algorithms (VQAs). The general VQA framework minimises:

$$\mathcal{F}(\boldsymbol\theta) = \langle\psi(\boldsymbol\theta)|\hat{H}|\psi(\boldsymbol\theta)\rangle$$

over parameters $\boldsymbol\theta$ of a parameterised quantum circuit $U(\boldsymbol\theta)$, with $|\psi(\boldsymbol\theta)\rangle = U(\boldsymbol\theta)|0\rangle^{\otimes n}$.

For QAOA: $\hat{H} = -\hat{C}$ (we minimise the negative expected cut) and $\boldsymbol\theta = (\boldsymbol\gamma, \boldsymbol\beta)$.

Adiabatic Motivation¶

The QAOA circuit can be seen as a Trotterised approximation of the quantum adiabatic algorithm (QAA). Starting from the ground state of $B = \sum_i X_i$ (uniform superposition), QAA slowly evolves:

$$H(t) = \left(1 - \frac{t}{T}\right)B + \frac{t}{T}C$$

For slow enough evolution ($T \to \infty$), the system remains in the ground state and reaches the MaxCut solution (adiabatic theorem). QAOA approximates this with $p$ discrete steps.

Why Graph Neural Networks?¶

Three properties make GNNs ideal for learning QAOA parameters:

  1. Structural inductive bias: QAOA performance depends on graph topology (degree sequence, cycle structure, edge density) — exactly what GCNs naturally encode.
  2. Permutation invariance: Relabelling graph nodes doesn't change the MaxCut value or optimal angles — GCN with global pooling has this invariance built in.
  3. Generalisation: A GNN trained on $n \in [6,10]$ graphs generalises to $n=12$ with <5% quality loss due to the local message-passing structure.

Part 1: QAOA — MaxCut Variational Parameter Search ¶

MaxCut Hamiltonian¶

Given graph $G = (V, E)$ with $|V| = n$, $|E| = m$, the MaxCut cost Hamiltonian is:

$$\hat{C} = \frac{1}{2}\sum_{(u,v)\in E}(I - Z_u Z_v)$$

where $Z_u$ is the Pauli-Z operator on qubit $u$, acting as identity on all other qubits.

The eigenvalue of $\hat{C}$ on a computational basis state $|x\rangle = |x_1\cdots x_n\rangle$ equals the number of cut edges when $0 \rightarrow +1$ and $1 \rightarrow -1$:

$$\hat{C}|x\rangle = C(\mathbf{z}(x))|x\rangle, \quad C(\mathbf{z}) = \sum_{(u,v)\in E}\frac{1 - z_u z_v}{2}$$

QAOA Circuit at Depth $p=1$¶

Initial state: $|+\rangle^{\otimes n} = H^{\otimes n}|0\rangle^{\otimes n} = \frac{1}{\sqrt{2^n}}\sum_{x\in\{0,1\}^n}|x\rangle$

Cost unitary: $U_C(\gamma) = e^{-i\gamma\hat{C}} = \prod_{(u,v)\in E} e^{-i\frac{\gamma}{2}(I - Z_uZ_v)}$

Each factor $e^{-i\frac{\gamma}{2}(I - Z_uZ_v)}$ is implemented as: $$e^{i\gamma/2} \cdot e^{-i\frac{\gamma}{2}Z_uZ_v} = e^{i\gamma/2} \cdot R_{ZZ}(\gamma)_{uv}$$

Mixer unitary: $U_B(\beta) = e^{-i\beta\sum_i X_i} = \prod_{i=1}^n e^{-i\beta X_i} = \prod_{i=1}^n R_X(2\beta)_i$

Variational state: $$|\gamma, \beta\rangle = U_B(\beta)\,U_C(\gamma)\,|+\rangle^{\otimes n}$$

Objective: $$\langle C\rangle(\gamma, \beta) = \langle\gamma,\beta|\hat{C}|\gamma,\beta\rangle$$

For the demo graph with $n=6$, we also compute the exact MaxCut via exhaustive search over $2^n = 64$ bipartitions — this is only feasible for small $n$ and serves as the ground-truth benchmark for our QAOA approximation ratio.

              if ((mask >> u) & 1) != ((mask >> v) & 1))
    if cut > best_cut:

        best_partition = mask

print(f"Best partition: {format(best_partition, f'0{n}b')}  "

      f"S̄={[i for i in range(n) if not (best_partition>>i)&1]})")

# ── Classical QAOA optimisation: minimise -⟨C⟩(γ, β) via Nelder–Mead
def classical_optimize(n, edges, p, maxiter=300, seed=0):
    """
    Nelder–Mead minimisation of the negative QAOA expected cut.

    The parameter space is [0, π]^p × [0, π/2]^p.
    At p=1, this is a 2D unconstrained optimisation.
    """
    rng = np.random.default_rng(seed)

    def neg_exp(x):
        g, b = x[:p], x[p:]
        return -expected_cut(n, edges, qaoa_state(n, edges, g, b))

    x0 = rng.uniform(0, math.pi, size=2 * p)

    res = minimize(neg_exp, x0, method='Nelder-Mead',
                   options={'maxiter': maxiter, 'xatol': 1e-5, 'fatol': 1e-5})
    return res.x[:p], res.x[p:], -res.fun

t0 = time.perf_counter()
opt_g, opt_b, val_opt = classical_optimize(n, edges, p)
classical_time = time.perf_counter() - t0

print(f"\nNEQA Classical Optimisation (Nelder–Mead, p={p})")
print(f"  γ_opt   = {opt_g}")
print(f"  β_opt   = {opt_b}")
print(f"  ⟨C⟩_opt = {val_opt:.4f}  (exact MaxCut = {best_cut})")
print(f"  Approx ratio: {val_opt/best_cut:.4f}")

Part 2: GNN Parameter Prediction ¶

From Linear Search to Neural Amortisation¶

Classical Nelder–Mead runs a fresh optimisation for every new graph instance — $O(100)$ statevector evaluations, each $O(2^n)$. This is impractical for:

  • Online workloads: thousands of MaxCut instances per second
  • Large graphs: each evaluation is expensive
  • NISQ hardware: circuit repetitions are limited by device time

Amortised inference trains the GNN once on a synthetic distribution of graphs, then predicts optimal angles in a single forward pass at inference.

GNN Architecture: SimpleGCN¶

Input:

  • Node features $X \in \mathbb{R}^{n \times 1}$: degree of each node $X_i = \sum_j \hat{A}_{ij}$
  • Augmented adjacency $\hat{A} = A + I_n$: self-loops included

Layer $l$ update rule (symmetric normalisation, Kipf & Welling 2017): $$\mathbf{H}^{(l+1)} = \text{ReLU}\!\left(\hat{D}^{-1/2}\hat{A}\hat{D}^{-1/2}\,\mathbf{H}^{(l)}\,\mathbf{W}^{(l)}\right)$$

Global mean pooling: $$\bar{\mathbf{h}} = \frac{1}{n}\sum_{i=1}^n \mathbf{H}^{(L)}_i \in \mathbb{R}^{d_h}$$

Output head: $$[\hat\gamma_1, \hat\beta_1] = \bar{\mathbf{h}}\,\mathbf{W}_{\rm out} \in \mathbb{R}^{2p}$$

Training loss (end-to-end, differentiating through the statevector simulator): $$\mathcal{L}_{\rm GNN}(\Theta) = \mathbb{E}_{G \sim \mathcal{G}}\left[-\langle C\rangle\!\left(f_\Theta(G)\right)\right]$$

Why Degree Features Are Sufficient¶

In QAOA on $k$-regular graphs, the optimal $(\gamma^*, \beta^*)$ depends only on $k$ and $m/n$. For irregular graphs, the degree sequence provides the dominant structural information. More expressive features (triangle count, $k$-core decomposition) improve quality for heterogeneous graph distributions.

Part 3: Expected-Cut Landscape & 2D Parameter Heatmap ¶

Understanding the QAOA Energy Landscape¶

The function $\langle C\rangle(\gamma, \beta)$ defines a 2D energy landscape over the variational parameter space $[0,\pi] \times [0,\pi/2]$. Key properties:

  • Periodicity: $\langle C\rangle$ is periodic with period $\pi$ in $\gamma$ and $\pi/2$ in $\beta$
  • Smoothness: the landscape is infinitely differentiable (composed of trigonometric functions)
  • Multi-modality: for $p \geq 2$, the landscape can have many local optima
  • Concentration: for large $n$, typical instance values concentrate around their expected value (Brandao et al., 2018)

Expected-Cut Formula for a Triangle $(K_3)$¶

For a 3-node complete graph $K_3$ at depth $p=1$, the closed form is:

$$\langle C\rangle(\gamma, \beta) = \frac{3}{2} - \frac{3}{4}\!\left[\cos(2\gamma)\sin(2\beta)^2\sin(\gamma)\right]$$

(derived from the analytical $p=1$ formula of Wang et al., 2018)

For larger irregular graphs, we use the numerical statevector approach.

We visualise where on the 2D landscape the GNN lands its prediction $(\hat\gamma, \hat\beta)$ relative to the classical optimum $(\gamma^*, \beta^*)$. A good GNN prediction should land near a high-value region of the landscape, even if not exactly at the peak.

Part 4: Biomedical GCN — Graph-Structured Cancer Diagnosis ¶

Dataset: Wisconsin Diagnostic Breast Cancer (WDBC)¶

  • Samples: 569 patients from the University of Wisconsin
  • Features: 30 real-valued measurements from FNA biopsy images
  • Classes: malignant (212) vs. benign (357)
  • Source: UCI ML Repository, Wolberg & Mangasarian (1994)

Why a Graph-Based Approach?¶

A standard MLP applies an identical function to each patient independently. The k-NN GCN additionally:

  1. Aggregates information from similar patients (k nearest neighbours)
  2. Propagates across the similarity manifold — a borderline malignant case will receive information from its clearly-malignant neighbours
  3. Learns manifold-aware representations that capture inter-sample correlations

GCN Architecture¶

$$\text{Input: }\mathbf{X} \in \mathbb{R}^{n\times 30}, \quad \hat{A} \in \mathbb{R}^{n\times n} \text{ (k-NN adj + self-loops)}$$

$$\mathbf{H}^{(1)} = \text{ReLU}(\hat{A}_{\rm norm}\,\mathbf{X}\,\mathbf{W}^{(0)}_{30 \to 64})$$

$$\mathbf{H}^{(2)} = \text{ReLU}(\hat{A}_{\rm norm}\,\mathbf{H}^{(1)}\,\mathbf{W}^{(1)}_{64 \to 32})$$

$$\hat{y}_i = \text{softmax}\!\left(\mathbf{H}^{(2)}_i\,\mathbf{W}^{(2)}_{32 \to 2}\right)$$

Symmetric Normalisation vs. Random Walk¶

The code uses symmetric normalisation for the biomedical GCN:

$$\hat{A}_{\rm norm} = \hat{D}^{-1/2}\hat{A}\hat{D}^{-1/2}$$

This symmetrises the spectrum and provides better gradient stability than single-sided row normalisation.

class BioGCN(nn.Module):
    """
    Two-layer GCN for transductive node classification.

    Forward pass:
        H¹ = ReLU(A_norm · X · W⁰)         shape (n, hidden)
        H² = ReLU(A_norm · H¹ · W¹)        shape (n, hidden//2)
        logits = H² · W²                   shape (n, n_classes)

    Note: bias=False follows Kipf & Welling (2017) convention,
    which works well with batch normalisation but is also common without it.

    Args:
        in_dim    : input feature dimension (30 for WDBC)
        hidden    : first hidden layer dimension
        n_classes : number of output classes (2)
    """
    def __init__(self, in_dim: int, hidden: int, n_classes: int):
        super().__init__()
        self.fc1 = nn.Linear(in_dim,      hidden,    bias=False)
        self.fc2 = nn.Linear(hidden,      hidden//2, bias=False)
        self.clf = nn.Linear(hidden//2,   n_classes)
        # Optional: add dropout for regularisation
        self.drop = nn.Dropout(p=0.1)

    def forward(self, A: torch.Tensor, X: torch.Tensor) -> torch.Tensor:
        h = F.relu(A @ self.fc1(X))      # Layer 1: graph conv + ReLU
        h = self.drop(h)
        h = F.relu(A @ self.fc2(h))      # Layer 2: graph conv + ReLU
        return self.clf(h)               # Classifier head (logits)

Part 5: Reading the Evaluation Dashboard ¶

The 6-panel dashboard captures every aspect of the BioGCN performance:

Row 1 — Statistical Performance¶

Confusion matrix (top-left):

  • Diagonal = correct predictions (TP + TN)
  • Off-diagonal = errors: FN (bottom-left, missed cancers) and FP (top-right, false alarms)
  • PPV and NPV annotations give the positive/negative predictive values

ROC curve (top-middle):
A perfect classifier hugs the top-left corner (AUC=1). The filled area under the curve represents the probability that a randomly chosen malignant patient scores higher than a randomly chosen benign patient. For WDBC, most GCN models achieve AUC ≥ 0.98.

Training dynamics (top-right):

  • Train and test accuracy should converge and remain close (no large gap = no overfitting)
  • The loss (green dashed, right axis) should decrease monotonically
  • If test accuracy drops while train accuracy rises → overfitting → add dropout or weight decay

Row 2 — Geometric Insights¶

PCA true labels (bottom-left):
Reveals the intrinsic geometry of the WDBC data. Blue cluster = benign, red cluster = malignant. Overlap in PCA space ≠ hard classification problem, because PCA drops ~35% of variance.

PCA predictions (bottom-middle):
Gold × markers indicate misclassified samples. Most errors appear in the overlap region between the two PCA clusters — these are the biologically ambiguous cases.

Feature importance (bottom-right):
A linear logistic regression trained on the same data reveals which features the linear boundary uses most. The features with high weight (e.g., worst concave points, worst perimeter) reflect well-established pathological markers for malignancy.

Synthesis: How QAOA and GNN Connect ¶

The Unified Hybrid Pipeline¶

Both components of this notebook use Graph Neural Networks, but for different purposes:

Component Graph Node feature GNN output Task
QAOA GNN MaxCut instance degree+1 $(\hat\gamma, \hat\beta)$ Parameter prediction
BioGCN Patient similarity clinical features class logits $\hat{y}_i$ Diagnosis

Integration: QAOA ↔ Biomedical GCN¶

The hybrid pipeline enables novel workflows:

  1. GCN-guided cohort partitioning:
    The BioGCN produces patient embeddings $\mathbf{h}_i$. Define a similarity graph from these embeddings and apply QAOA-MaxCut to partition patients into two cohorts maximising between-group dissimilarity — a quantum-accelerated version of balanced clustering.

  2. Feature selection via QAOA:
    Build a feature correlation graph (nodes = features, edges = high correlation $> 0.9$). Apply QAOA to find the maximum independent set — a set of minimally correlated features for downstream modelling.

  3. GNN warm-starting QAOA:
    Use GCN feature embeddings as intermediate representations in the QAOA state initialisation, replacing the uniform superposition $|+\rangle^{\otimes n}$ with a data-driven initial state.

Mathematical Connection¶

For QAOA, the GNN learns a map $f_\Theta: G \to \mathbb{R}^{2p}$ from graph to optimal QAOA angles. This map generalises over the graph distribution.

For BioGCN, the GNN learns a map $g_\Phi: (\mathcal{G}, \mathbf{X}) \to \mathbb{R}^{n \times C}$ from a labelled patient graph to node-level class predictions.

Key shared principle: both $f_\Theta$ and $g_\Phi$ are permutation-equivariant with respect to node relabelling, and inductive (generalise to new graphs without retraining) when trained on a diverse distribution.

Future Directions ¶

Near-term Research (NISQ)¶

  • QAOA for portfolio optimisation: Pharmaceutical drug trial design as a QUBO — select the $k$ most diverse compounds from $n$ candidates
  • Variational quantum eigensolver (VQE) for molecular Hamiltonians — drug binding energy estimation
  • Quantum-enhanced GNN: Replace innermost linear layers with parameterised quantum circuits (data re-uploading technique)

Medium-term (early FTQC)¶

  • Grover-accelerated graph search: $O(\sqrt{2^n})$ evaluation of the MaxCut objective
  • HHL algorithm for linear systems in GNN training: speedup for dense $\tilde{A}\mathbf{H}\mathbf{W}$

Long-term (fully FTQC)¶

  • Quantum machine learning speedups for kernel methods (QSVM)
  • Quantum backpropagation for GNN training on quantum hardware
